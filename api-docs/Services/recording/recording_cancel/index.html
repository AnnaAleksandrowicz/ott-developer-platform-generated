Kaltura OTT APIrecording.cancel

Cancel a previously requested recording. Cancel recording can be called for recording in status Scheduled or Recording Only

Try it out
post https://tvpapi-us-preprod.ott.kaltura.com/api_v3/service/recording/action/cancel

Parameters

  • body
  • header
body:object  required
SchemaExample
{
  "ks": "",
  "version": "3.6.1542.28904",
  "id": 0
}object
id:integer
 must also match all of:
object:OTTRequest

Responses

  • 200
  • Errors

Success

SchemaExample
{
  "assetId": 0,
  "createDate": 0,
  "id": 0,
  "isProtected": true,
  "status": "SCHEDULED",
  "type": "SINGLE",
  "updateDate": 0,
  "viewableUntilDate": 0
}object:KalturaRecording
assetId:integer
Kaltura unique ID representing the program identifier


createDate:integer
readOnly


Specifies when was the recording created. Date and time represented as epoch.


id:integer
readOnly


Kaltura unique ID representing the recording identifier


isProtected:boolean
readOnly


Specifies whether or not the recording is protected


status:string
Enum: SCHEDULED, RECORDING, RECORDED, CANCELED, FAILED, DELETED
readOnly


Enum Type: KalturaRecordingStatus


Recording state: scheduled/recording/recorded/canceled/failed/does_not_exists/deleted


type:string
Enum: SINGLE, SEASON, SERIES
readOnly


Enum Type: KalturaRecordingType


Recording Type: single/season/series


updateDate:integer
readOnly


Specifies when was the recording last updated. Date and time represented as epoch.


viewableUntilDate:integer
readOnly


Specifies until when the recording is available for viewing. Date and time represented as epoch.
